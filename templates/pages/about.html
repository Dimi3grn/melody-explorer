À propos de MelodyExplorer

Aperçu du projet

MelodyExplorer est une application web qui permet aux utilisateurs de découvrir et d'explorer la musique en utilisant l'API Spotify. Ce projet a été créé dans le cadre du devoir "Groupie Tracker", qui exigeait la création d'une application web consommant une API REST et implémentant diverses fonctionnalités.

Fonctionnalités

  • Système de recherche : Recherchez des artistes, des albums et des pistes.
  • Système de filtrage : Filtrez la musique par type, genre, popularité et année de sortie.
  • Pagination : Parcourez facilement de grandes collections de musique.
  • Favoris : Enregistrez votre musique préférée pour un accès rapide.

Technologies utilisées

  • Backend : Go (Golang)
  • Frontend : HTML, CSS, JavaScript
  • API : API Web Spotify
  • Stockage de données : Fichiers JSON

FAQ du projet

Comment avez-vous décomposé le projet ? Quelles ont été les phases clés ?

J'ai décomposé le projet en ces phases clés :

  1. Exploration de l'API : J'ai d'abord pris le temps de comprendre l'API Spotify, ses points de terminaison, son mécanisme d'authentification et ses structures de données. J'ai utilisé Postman pour tester différents points de terminaison et me familiariser avec les formats de réponse.
  2. Planification de l'architecture : J'ai conçu l'architecture de l'application, en décidant de la structure des répertoires, de l'organisation des fichiers et de la façon dont les différents composants interagiraient entre eux.
  3. Développement du backend : J'ai implémenté le backend Go, en commençant par le client Spotify pour gérer les interactions avec l'API, puis j'ai créé les gestionnaires, les middlewares et les routes.
  4. Modèles frontend : J'ai conçu et implémenté les modèles HTML pour différentes pages, assurant une mise en page et une expérience utilisateur cohérentes.
  5. Style et interactivité : J'ai ajouté des styles CSS et des fonctionnalités JavaScript pour rendre l'application visuellement attrayante et interactive.
  6. Tests et améliorations : J'ai testé en profondeur toutes les fonctionnalités, corrigé les bugs et apporté des améliorations basées sur des considérations d'utilisabilité.

Comment avez-vous géré la répartition des tâches pour le projet ?

Comme il s'agissait d'un projet individuel, je n'ai pas eu à distribuer les tâches entre les membres de l'équipe. Cependant, j'ai organisé mon propre travail en :

  • Créant un tableau de tâches avec des tâches et des priorités clairement définies
  • Regroupant les tâches connexes pour maintenir l'attention sur des composants spécifiques
  • Implémentant une fonctionnalité à la fois, en m'assurant qu'elle fonctionne correctement avant de passer à la suivante
  • Révisant régulièrement mes progrès par rapport aux exigences du projet

Comment avez-vous géré votre temps ? Avez-vous défini des priorités ?

La gestion du temps était cruciale pour réussir ce projet. Mon approche comprenait :

  • L'établissement d'un calendrier de projet avec des jalons clairs
  • La priorisation des fonctionnalités essentielles par rapport aux améliorations esthétiques
  • La définition d'objectifs quotidiens pour maintenir des progrès constants
  • L'allocation de temps supplémentaire pour les fonctionnalités complexes et les problèmes potentiels
  • L'évaluation régulière des progrès et l'ajustement des priorités si nécessaire

J'ai priorisé les exigences fondamentales (recherche, filtrage, pagination, favoris) avant d'ajouter des fonctionnalités ou des améliorations supplémentaires.

Quelle stratégie avez-vous adoptée pour la documentation ?

Ma stratégie de documentation comprenait :

  • Le maintien de commentaires de code détaillés pour les fonctions et la logique complexe
  • La création d'un fichier README complet avec des instructions d'installation et des explications de fonctionnalités
  • La documentation des points de terminaison API et de leur utilisation
  • La prise de notes sur les défis rencontrés et les solutions mises en œuvre
  • La création de diagrammes pour visualiser l'architecture de l'application
  • La documentation des points de terminaison de l'API Spotify utilisés et leur rôle dans l'application

Cette approche a permis de s'assurer que le projet serait facile à comprendre et à maintenir, tant pour moi que pour toute autre personne qui pourrait travailler avec le code.

Informations sur l'API

Ce projet utilise les points de terminaison suivants de l'API Spotify :

  • /search : Rechercher des artistes, des albums et des pistes
  • /artists/{id} : Obtenir des informations détaillées sur un artiste
  • /artists/{id}/albums : Obtenir les albums d'un artiste
  • /albums/{id} : Obtenir des informations détaillées sur un album
  • /albums/{id}/tracks : Obtenir les pistes d'un album
  • /tracks/{id} : Obtenir des informations détaillées sur une piste
  • /recommendations/available-genre-seeds : Obtenir les genres disponibles
  • /recommendations : Obtenir des recommandations de pistes
